# 低 Token 提问模板（个人版）

## 使用方式
- 每次只处理一个任务，尽量开新会话。
- 先限定文件范围，再描述问题。
- 没必要的信息不要给，避免上下文膨胀。

## 通用模板（复制即用）
修改刷新为重置日期而不是重新请求

请你直接生成让cursor对话修改代码的最省token的指令。
例如下面模板
```
【任务类型】修Bug/功能开发/重构/代码审查/文档编辑/提问
【核心目标/问题】一句话说清可落地的最终结果
【明确前提&假设】列清前置判断，不确定项直接提问
【只允许查看】<文件1>,<文件2>（约3-5个，无则当前打开文件/自行商榷减少引用）
【不要做】不要全项目扫描；不要改无关代码；不要跑全量构建，不要自动执行全量测试；不要循环修复
【已知信息】报错/现象/复现步骤/已有能力（最多5行，极简表述）
【验收标准】可量化、可验证的成功条件（拒绝“能用就行”）
【输出要求】先给核心结论，再给修改点；代码改动尽量小
【如果是提问】给出指导意见后再回复这个模板
```
